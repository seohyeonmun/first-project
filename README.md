1. 출근 시간대 빠른 정보 확인 필요

- 기획 의도  
  호출어 + 음성  질문만으로 버스 도착 정보 제공

## 4. 주요 기능 
### KWS (openWakeWord)

'교통봇' 호출어를 감지하면 다음 동작으로 넘어갑니다.

- 직접 녹음한 200개의 호출어 데이터를 기반으로 데이터 증강을 수행하여 약 1000개의 학습 데이터를 구축했습니다.
- openWakeWord 모델을 파인튜닝하여 호출어 인식 성능을 향상했습니다.
- 오탐을 줄이기 위해 유사 발음 단어들을 네거티브 데이터로 추가 학습했습니다.
- 연속 4회 감지 방식으로 안정성을 높였습니다.
- 호출어 감지 score 0.97을 달성했습니다.


### STT (Whisper-Base) + 공공데이터포털 OpenAPI

호출어 감지 후 사용자 음성을 텍스트로 변환하고, 광주광역시 BIS API를 활용해 실시간 버스 도착 정보를 안내합니다.

- 사용자 질문에서 버스 번호와 정류소 이름을 추출합니다.
- Levenshtein Ratio 유사도 비교를 적용하여 정류소명을 잘못 인식해도 유사한 정류소를 탐색할 수 있도록 구현했습니다.
- 동일한 이름의 정류소가 상행·하행으로 나뉘는 경우를 고려하여 정류소 + 종점 형태로 안내합니다.


### TTS (GPT-SoVITS-v2Pro)

생성된 답변을 신뢰감 있는 여성 아나운서 스타일 음성으로 출력합니다.

- 자연스러운 한국어 발화와 추론 속도의 균형을 고려해 GPT-SoVITS-v2Pro를 사용했습니다.
- API 서버를 도입해 모델을 메모리에 상주시켜 즉시 추론 가능하도록 최적화했습니다.
- 로컬 추론 시 약 20초 소요되던 응답 시간을 API 서버 도입 후 1~3초 수준으로 단축했습니다.
